# GP0102 游戏埋点 Skill 使用说明

## 快速开始

1. 复制本文件夹到 `<项目>/.cursor/skills/gp0102-game-tracking/`
2. 对话：`@gp0102-game-tracking 帮我出 XX 功能的埋点方案`
3. 描述功能需求，agent 会自动选择表和字段，输出埋点方案

## 使用示例

```
@gp0102-game-tracking 帮我出宣战功能的埋点方案，需要记录宣战成功、取消宣战、消耗/返还黏土
```

```
@gp0102-game-tracking 硕果积分活动需要记录每小时积分变动和档位奖励领取
```

```
@gp0102-game-tracking 新增一个前哨营地的战斗类型，怎么打点
```

## 红线约束

- 不能新增日志表（如 log_outpost_change）
- 不能新增字段（如在现有表中加列）
- 新数据通过 extend_1~5（JSON）承载
- 新行为通过 action/step_type/step_id 现有字段的值区分

## 方案输出格式

每行一个埋点点位，包含：

| 列 | 说明 |
|---|---|
| 属性类别 | 玩家基本信息 / 行为数据 / 道具信息 等 |
| 上报字段 | 具体字段名 |
| 字段中文名 | 中文说明 |
| 字段类型 | varchar / bigint / integer / boolean |
| 属性描述/属性值/备注 | 取值规范、JSON 格式、关联关系 |
