# Git 工作流 Skill 使用说明

## 快速开始

1. 复制本文件夹到 `<项目>/.cursor/skills/git-workflow/`
2. 对话：`@git-workflow 拉取最新代码`
3. 或：`@git-workflow 推送所有更改`

## 常用场景

### 拉取最新代码

```
@git-workflow 拉取远程最新代码
@git-workflow 同步远程仓库
@git-workflow pull
```

### 提交并推送

```
@git-workflow 把当前改动推送到 GitHub
@git-workflow 提交并推送
@git-workflow push
```

### 查看状态

```
@git-workflow 看下当前 Git 状态
@git-workflow 最近的提交记录
@git-workflow 查看某个文件的修改历史
```

### 分支管理

```
@git-workflow 创建新分支 feat/new-feature
@git-workflow 合并分支到 main
@git-workflow 删除已合并的分支
```

## 提交信息格式

```
<type>: <简短描述>
```

type 可选：

| type | 用途 |
|---|---|
| feat | 新功能 |
| fix | 修复 bug |
| docs | 文档变更 |
| style | 格式调整 |
| refactor | 重构 |
| chore | 工具/构建 |
| delete | 删除 |
| rename | 重命名 |

## 安全约束

- 禁止强制推送 (`git push --force`)
- 禁止提交密码/token
- 禁止硬回退 (`git reset --hard`)，除非用户明确要求
- 禁止修改 `.git/config`
