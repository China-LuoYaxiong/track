# 埋点验证工具使用说明

## 快速开始

### 1. 输入文件

- **埋点方案 Excel**：`副本-【odirouter】埋点方案与开发计划.xlsx`（已加入 `source/` 作为默认参考）
- **测试数据 CSV**：`odirouter_test_data.csv`（已加入 `source/` 作为默认参考）

### 2. 指定序号

| 格式 | 示例 | 说明 |
|------|------|------|
| 单个 | `35` | 验证序号 35 |
| 范围 | `20-35` | 验证 20 到 35 |
| 列表 | `20,25,30` | 指定多个序号 |
| 混合 | `20-25,30,35` | 范围 + 列表 |
| 全量 | `1-175` | 全部点位 |

### 3. 执行验证

对 AI 说：

```
验证序号107-110
副本-【odirouter】埋点方案与开发计划.xlsx
odirouter_test_data.csv
```

或命令行（假设文件已在当前目录，或使用 source/ 下的默认文件）：

```bash
python skills/tracking-validator/scripts/validate.py \
  "副本-【odirouter】埋点方案与开发计划.xlsx" \
  "odirouter_test_data.csv" \
  "1-175"
```

| 变量 | 值 | 适用列 |
|------|-----|--------|
| `--col-event-width` | 128px | 埋点事件 |

### 报告交互与布局

- **整表横向滚动**：列多时可左右拖动（`.table-wrap`）；表格 `width: max-content` + `<colgroup>` 固定列宽，避免列重叠
- **埋点信息换行**：长名称自动折行，不再撑开整行
- **埋点事件**：128px 宽，可自动换行
- **上报参数详情**：320px（埋点事件列 × 2.5），可自动换行
- **独立滚动区域**（固定高度，**横向 + 纵向**均可滚动，仅用于案例与错误/警告）：

| 区域 | CSS 类 | 列宽 | 高度 |
|------|--------|------|------|
| 入库案例 / 错误案例 | `.col-case` + `.scroll-box.case-box` | 510px | 180px |
| 入库错误原因或警告 | `.col-error` + `.scroll-box.error-box` | 510px | 120px |

三列列宽均为 `--col-case-width: 510px`。

- **案例 JSON 内容**：每个 `"参数": 值` 独占一行，行内不折行（`white-space: pre`），长内容由外层 `.scroll-box` 滚动查看

## 判定逻辑

### 取最新一条

研发可能多次改代码上报，验证时每个点位在所有候选记录中按 `st_event_time` **取最新一条**作为判定依据。

### 入库 vs 警告

| 情况 | 是否入库 | 报告展示 |
|------|----------|----------|
| 最新一条 `st_status=1`，参数完全符合方案 | ✅ | 无警告 |
| 最新一条 `st_status=1`，有多余参数 | ✅ | **警告**（仍算入库成功） |
| 最新一条 `st_status≠1` | ❌ | **错误原因** + 错误案例 |

### 错误原因 vs 警告

**错误原因**（导致未入库）：
- `st_error_info` 系统校验错误
- 必填参数缺失

**警告**（不阻断入库）：
```
警告：参数【ref_page_id、current_page_id、status】上报参数详情中不要求上报
```

## 常见问题

### Q: 为什么后端埋点（如 log_register_server）之前显示无数据？

Excel 中页面标识为空时，验证脚本**跳过** `current_page_name` 匹配，仅按事件名匹配。

### Q: 为什么 st_status=1 还有警告？

多余参数仅作提醒，**不影响入库判定**。建议更新 Excel 方案或精简上报字段。

### Q: 为什么有数据但显示未入库？

看**错误原因**（不是警告）：通常是系统拒绝（属性不在方案中）或缺少必填参数。

### Q: 如何区分前后端？

- `_client` 结尾 → 前端
- `_server` 结尾 → 后端

## 技术细节

### 匹配三要素

1. `st_event_name` = 埋点事件
2. `current_page_name` = 页面标识（**Excel 为空则跳过**）
3. 上报参数详情中的非动态固定值（如 `element_module=reward_task`）

### 系统预置参数（不参与校验）

`extend`、`adid`、`current_page_url`、`platform_type`
